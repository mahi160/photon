Frame 002 — the point

If mpv can play it, Photon plays it

Photon embeds mpv directly — in-process, via its render API, compositing straight into Photon's own window. GPU-rendered, with an automatic CPU fallback if a machine can't do that. Always direct play: the server never has to transcode. This is the whole reason Photon exists; everything else is the UI around it.

renderer: gpu · fallback: cpu (auto) · play: direct (always)

Frame 003 — what it does

Sign in, browse, hit play

  • Continue Watching, Recently Added Movies, Recently Added Shows on Home
  • Movies and TV Shows grids, every library merged
  • Instant local search plus server-side episode search
  • Audio and subtitle track switching; delay and styling for text subtitles
  • Picture-in-Picture, fullscreen, playback speed
  • Keyboard-first controls
  • Watch progress synced back to Jellyfin

Frame 005 — install

macOS, verified

Homebrew (recommended)

brew install --cask mahi160/photon/photon

Or download the .dmg

Ad-hoc signed, not notarized, so Gatekeeper needs a one-time fix (the cask above does this automatically): 

xattr -cr /Applications/Photon.app

Windows / Linux

.exe/.deb/.rpm/.AppImage builds are on Releases — they install and run, but video doesn't render yet (#27, help wanted). Releases are flagged pre-release on GitHub until that's fixed.
